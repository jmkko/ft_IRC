53 * @details
54 * - supported modes are channel modes
55 - `k` key
56 - `i` invite-only
57 - `l` member limit
58 - `t` restrict usage of TOPIC to operators

60 * - supported operations are `+` and `-`
61 * - modes requiring arguments are `k` (valid key), `l` (int from -1 to INT_MAX) abd `o` (valid username)
62 * sends a RPL_CHANNELMODEIS in case it is successful - and RPL_YOUREOPER (for +o)
63 * @remark parsing is less flexible than RFC suggests : if a mode requires a value, it should be provided after the modes at
64 the corresponding index. In other words, `+kl key 10` is accepted but `+k key +l 10` is rejected
65 * @remark If duplicate modes are present (as in `+kk key1 key2`) the last one prevails
